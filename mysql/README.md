# MySQL 数据库初始化说明

## 目录结构

```
mysql/
├── init/           # 数据库初始化脚本目录
│   ├── init.sql    # 数据库表结构定义（必须）
│   └── data.sql    # 示例数据（可选，当前为空）
└── data/           # 数据库数据目录（运行时自动生成，不应提交）
```

## 初始化说明

### Docker 部署方式

使用 Docker Compose 启动时，MySQL 容器会自动执行 `init/` 目录下的 SQL 文件来初始化数据库。

执行顺序：
1. `init.sql` - 创建数据库和表结构
2. `data.sql` - 插入示例数据（如果有）

### 本地开发方式

如果需要手动初始化数据库：

1. 启动 MySQL 服务
2. 执行初始化脚本：
   ```bash
   mysql -u root -p < mysql/init/init.sql
   ```

## 注意事项

- `mysql/data/` 目录是 MySQL 运行时生成的数据文件，包含实际的数据库数据
- 该目录不应提交到版本控制系统
- 首次启动 Docker 容器时，会自动创建该目录并初始化数据库
- 如果需要清空数据库，可以删除 `mysql/data/` 目录后重新启动容器

## 数据库信息

- **数据库名**: mydb
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **默认用户**: root
- **默认密码**: root（生产环境请修改）
